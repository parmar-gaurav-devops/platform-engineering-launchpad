# Contributing

Thank you for improving the launchpad.

1. Open an issue before large changes so the problem and success criteria are clear.
2. Keep pull requests focused and include documentation or examples for user-facing changes.
3. Run `npm test` before submitting scorecard changes.
4. Never include cloud credentials, account identifiers, or customer data in examples.

By contributing, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
